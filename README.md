# passwordmanager

## 1. Описание

**PasswordManager** — простая консольная утилита для хранения и управления вашими логинами и паролями.  
Все данные сохраняются в зашифрованном локальном файле `data.vault`, а ключ шифрования берётся из переменной окружения.

> ⚠️ Проект учебный. Не используйте его для хранения настоящих паролей в публичных репозиториях.

---

## 2. Стек технологий

- **Язык:** Go 1.24.1  
- **Библиотеки:**
  - [`github.com/fatih/color`](https://github.com/fatih/color) — цветной вывод в консоли  
  - [`github.com/joho/godotenv`](https://github.com/joho/godotenv) — загрузка `.env` файла с переменными окружения  
- **Хранилище:** локальный файл `data.vault` (зашифрованный JSON)
- **ОС:** Windows (PowerShell 7.5.2), Linux, macOS

## 3. Инструкция по установке

### Требования
- Установленный **Go** (версия 1.24.1 или выше)  
- **PowerShell 7.5.2** (или любая консоль)  
- **Git**, если клонируете репозиторий

### Установка
1. Клонируйте репозиторий:
   ```powershell
   git clone https://github.com/sq1er/passwordmanager.git
   cd passwordmanager
   ```

2. Создайте файл `.env` в корне проекта и добавьте в него ваш ключ шифрования:
   ```env
   KEY=your-32-byte-secret-key
   ```
   **Важно:** не публикуйте `.env` в открытый доступ!  
   Ключ должен иметь длину **16, 24 или 32 байта** (для AES-128/192/256).

   Альтернатива — задать ключ прямо в PowerShell:
   ```powershell
   $env:KEY = "your-32-byte-secret-key"
   ```

3. Установите зависимости:
   ```powershell
   go mod tidy
   ```

## 4. Инструкция по использованию

### Запуск
Запуск без сборки:
```powershell
go run .
```

Сборка и запуск исполняемого файла:
```powershell
go build 
.\passwordmanager.exe
```

---

### Главное меню

После запуска появится меню:

```
1. Вывести все аккаунты
2. Добавить аккаунт
3. Найти аккаунт по URL
4. Найти аккаунт по логину
5. Удалить аккаунт
6. Выйти
Выберите вариант:
```

---

### Примеры работы

**Добавление аккаунта**
```
Введите логин: mylogin
Нажмите 1 чтобы ввести пароль,
enter чтобы сгенерировать пароль: 1
Введите пароль от аккаунта: mypassword123
Введите URL аккаунта: https://example.com
Аккаунт добавлен
```

**Поиск по URL**
```
Введите URL для поиска: https://example.com
Найден аккаунт:
Login: mylogin
Password: mypassword123
Url: https://example.com
```

**Удаление аккаунта**
```
Введите URL аккаунта для удаления: https://example.com
Удалено
```

## Примечания

- Все данные сохраняются в файл `data.vault` в корне проекта.  
- Ключ шифрования берётся из `.env` или переменной окружения `KEY`.  
- Файл `.env` уже добавлен в `.gitignore` — **не загружайте его в GitHub!**
- При первом запуске создаётся пустое зашифрованное хранилище.
- Если при запуске видите сообщение:
  ```
  panic: Не передан параметр KEY в переменные окружения
  ```
  — значит, ключ не был найден. Убедитесь, что `.env` создан или переменная окружения установлена.
